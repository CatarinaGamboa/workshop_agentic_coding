What Does This Repository Do?

Barista is an AI-agent system that automatically synthesizes typestate specifications for Java classes. A typestate tracks which methods are valid to call based on an object's current state — for example, you can't read() from a closed stream.

This repository stores the results and logs from running Barista on 35 Java standard library classes, 5 independent times each (175 total runs). It serves as a dataset for analyzing AI agent performance in specification synthesis.

Agent Pipeline

Each run spawns an orchestrator that delegates to specialized sub-agents:

orchestrator
Coordinates the overall workflow, spawns sub-agents, tracks tasks
protocol-analyzer
Analyzes the Java class to build a state machine (DFA) of valid method transitions
java-test-completer
Generates test cases to validate the state machine against actual Java behavior
liquidjava-spec-writer
Writes the LiquidJava refinement specification file from the state machine
liquidjava-verifier
Verifies the generated specification compiles and passes LiquidJava checks

Key Concepts

Typestate — A protocol that defines which methods are valid in each state. Example: a Socket has states like unconnected, connected, inputshutdown, outputshutdown, bothshutdown, and closed. You can only call connect() from the unconnected state.
LiquidJava — A verification framework that checks Java code against typestate refinements using annotations: @StateSet (declares states), @StateRefinement (method pre/post-conditions on states), and @Refinement (value constraints).
Refinements File — A *Refinements.java interface that declares all states and method contracts for a Java class.
Complexity — Each class is rated by two human reviewers on specification difficulty: none (no protocol), simple (0 factors), moderate (1 factor), complex (2+ factors). Factors include branching transitions, scale, multi-state sets, state-dependent values, and encoding workarounds.
